# Contributing

Thanks for considering contributing to this awesome list!

## Guidelines

1. **The project must be open-source** — OSI-approved license (MIT, Apache-2.0, GPL, etc.)
2. **The project must be self-hostable** — runs on your own infrastructure
3. **The project must be actively maintained** — or in a stable, usable state
4. **No hosted-only services** — this list is for software you run yourself

## Format

Each entry should follow this format:

```
### Project Name

[![GitHub stars](https://img.shields.io/github/stars/owner/repo?style=flat-square)](https://github.com/owner/repo)

Short description of what the project does and why it matters.

- **Fees:** (transaction fees or 0%)
- **Custody:** Non-custodial / Custodial
- **Focus:** (primary use case)
- **Deployment:** Docker, binary, script
- **Stack:** (tech stack)
- **License:** (license name)

[Source Code](https://github.com/owner/repo) · [Website](https://example.com)
```

## Adding a project

1. Fork this repository
2. Add your project to the appropriate category in `README.md`
3. Submit a pull request

## Removing a project

Open an issue if a project is no longer maintained, archived, or no longer self-hosted.

## Updating

Open an issue or PR to update outdated information (GitHub stars, new features, license changes, etc.).
